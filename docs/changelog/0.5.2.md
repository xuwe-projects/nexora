# Nexora 0.5.2

发布日期：2026-07-18

本补丁版本修复框架主 Shell 的 Sidebar 导航实现：恢复使用 `gpui-component` 官方
`Sidebar`、`SidebarGroup`、`SidebarMenu` 与 `SidebarMenuItem` 组件，避免导航项文本被手写
按钮布局居中。公开 API、配置、数据库迁移和依赖 revision 均保持与 0.5.1 兼容，因此严格按
Semantic Versioning 递增 PATCH。

## Fixed

- 框架默认 Sidebar 导航重新使用官方 `gpui-component::sidebar` 组件族渲染 section、目录和
  Feature 项。导航文字恢复官方 `SidebarMenuItem` 的左对齐行为，活动态、hover 和展开目录
  仍由组件库统一处理。— 处理人：[@Jixiangup](https://github.com/Jixiangup)
- 保留应用自定义 `SidebarHeader` 与 `SidebarFooter` 的插槽边界，Shell 只托管结构、间距和
  分隔线，不向自定义内容注入 hover、selected、圆角、cursor 或点击语义。— 处理人：
  [@Jixiangup](https://github.com/Jixiangup)
- 增加源码契约回归测试，锁定主 Shell 必须使用官方 Sidebar、Group、Menu 和 MenuItem，防止
  后续再次用 Button 或手写 flex 导航替代官方组件。— 处理人：
  [@Jixiangup](https://github.com/Jixiangup)
- 文档构建脚本改为通过 Node 显式启动 VitePress，避免 Bun 直接执行 `vitepress` bin 时在本地
  发布环境中被 SIGKILL。— 处理人：[@Jixiangup](https://github.com/Jixiangup)

## Issue 与 PR

- Issue：无。本版本修复来自 iMES 桌面接入时发现的 Sidebar 导航项对齐问题。
- PR：无；由默认分支直接修复并发布。

## 兼容性

- 无破坏性变更，无需修改 Feature、NavigationGroup、SidebarHeader 或 SidebarFooter 声明。
- HTTP API、OpenAPI 路径、ZITADEL 集成、数据库 schema 与迁移历史均未变化。
- `gpui`、`gpui-component` 与 assets revision 未变化。
- 文档构建产物内容不变；仅调整 `docs/package.json` 的 build script 启动方式。

## 升级

把 Nexora tag 从 `v0.5.1` 更新为 `v0.5.2` 并重新锁定依赖即可。不需要运行数据库迁移或更改
配置。回滚只需恢复 `v0.5.1` 依赖，但会重新引入默认 Sidebar 导航文字居中的视觉问题。
