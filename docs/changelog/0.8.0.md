# Nexora 0.8.0

发布日期：2026-07-19

本 early-alpha 版本为桌面 CRUD 资源管理页面沉淀标准 Panel 组件，并修复默认 Account 用户、
角色管理页面的若干布局与弹窗问题。由于 GPUI 与 gpui-component 仍通过固定 git revision
使用，Nexora 继续发布 `0.x` prerelease 版本，不进入 `1.x` 稳定通道。

## Added

- 新增 `nexora::desktop::{CrudPanel, CrudPanelToolbar}`，用于标准 CRUD Feature Panel 的标题
  摘要、刷新按钮、可选筛选/操作区，以及默认填满剩余高度的数据主体。工具栏没有筛选和操作时
  会自动省略整张卡片。— 处理人：[@Jixiangup](https://github.com/Jixiangup)
- 新增 `nexora::desktop::TableHeaderCell`，用于 DataTable 表头默认水平、垂直居中，并支持
  `.left()`、`.center()`、`.right()` 或完全自定义表头元素。— 处理人：
  [@Jixiangup](https://github.com/Jixiangup)
- 新增 UI crate 测试，锁定 CRUD Panel 空工具栏省略、工具栏内容判断和表头默认居中/可自定义
  行为。— 处理人：[@Jixiangup](https://github.com/Jixiangup)

## Changed

- 默认用户管理页面改用 `CrudPanel` 组织标题摘要、刷新、创建用户操作和用户表格；用户表格主体
  继续填满剩余高度并在 DataTable 内部滚动和无限加载。— 处理人：
  [@Jixiangup](https://github.com/Jixiangup)
- 更新生成项目的 `.agents/skills`，要求符合标题摘要、刷新、筛选/操作、数据主体结构的标准
  CRUD Panel 默认使用 `CrudPanel` / `CrudPanelToolbar`，并建议 DataTable 表头使用
  `TableHeaderCell`。— 处理人：[@Jixiangup](https://github.com/Jixiangup)

## Fixed

- 修复默认用户管理表格用户列内容显示不全的问题，扩大用户列宽度并让名称、ID 在受限宽度内
  稳定截断。— 处理人：[@Jixiangup](https://github.com/Jixiangup)
- 修复用户表格状态列过宽的问题，状态列改为按短状态内容固定宽度展示，不再占用整列剩余空间。
  — 处理人：[@Jixiangup](https://github.com/Jixiangup)
- 修复默认用户管理页面外层与 DataTable 之间的双层滚动问题，用户 Feature 关闭 Shell 外层
  content scroll，由 DataTable 自己管理滚动视口。— 处理人：
  [@Jixiangup](https://github.com/Jixiangup)
- 修复默认用户、角色创建和编辑浮层没有覆盖当前 Feature Panel 的问题，Account 默认浮层现在
  使用稳定的 panel-scoped overlay。— 处理人：[@Jixiangup](https://github.com/Jixiangup)
- 修复默认角色与权限页面刷新按钮图标与用户管理页面不一致的问题，统一使用 `rotate-ccw.svg`。
  — 处理人：[@Jixiangup](https://github.com/Jixiangup)

## Issue 与 PR

- Issue：无。本版本来自默认 Account 管理体验与标准 CRUD Panel 抽取需求。
- PR：无；按发布要求由默认分支直接提交并发布。

## 兼容性与破坏性变更

- 破坏性变更：无。
- 新增的 `CrudPanel`、`CrudPanelToolbar` 与 `TableHeaderCell` 是向后兼容公开 API；已有页面
  可以继续使用自定义布局。
- HTTP API、OpenAPI 路径、ZITADEL 集成、数据库 schema、迁移历史、`gpui` revision 与
  `gpui-component` revision 均未变化。
- 本版本仍是 `0.x` prerelease。GPUI 与 gpui-component 正式发布 crates 之前，Nexora 不进入
  `1.x` 稳定发布线。

## 升级

1. 把 Nexora tag 从 `v0.7.0` 更新为 `v0.8.0` 并重新锁定依赖。
2. 已有项目建议同步新脚手架中的 `.agents/skills` CRUD Panel 规则；新建项目会自动包含该规则。
3. 符合标题摘要、刷新、筛选/操作、数据主体结构的资源管理页，建议改用
   `nexora::desktop::{CrudPanel, CrudPanelToolbar}`；DataTable 表头建议使用
   `nexora::desktop::TableHeaderCell`。
4. 升级后建议在本地执行一次 `cargo clean`，再重新 `cargo check` 或启动应用，避免旧的 GPUI
   构建产物干扰排查。
5. 不需要运行数据库迁移或修改服务端配置。回滚到 `v0.7.0` 会移除新增 CRUD Panel 组件，但不会
   影响未使用这些组件的应用。
