# Nexora 0.1.1

发布日期：2026-07-17

本版本补齐 Account 全栈默认能力，并重新划分 Nexora 框架与宿主应用的资源所有权边界。

- 增加默认 Setup 初始化、ZITADEL 人类用户选择与系统超级管理员绑定。— 处理人：
  [@Jixiangup](https://github.com/Jixiangup)
- 增加用户、角色、权限定义和授权管理接口，以及桌面端默认管理页面。— 处理人：
  [@Jixiangup](https://github.com/Jixiangup)
- 增加可自定义应用名称、版本和 Logo 的登录品牌，登录失败通知支持复制 request ID。— 处理人：
  [@Jixiangup](https://github.com/Jixiangup)
- 增加 VitePress/Bun 中英文文档与 GitHub Pages 部署。— 处理人：
  [@Jixiangup](https://github.com/Jixiangup)
- 生成项目附带 Logo、注释配置、项目 Rules 和轻量 Feature 组件化 Skills。— 处理人：
  [@Jixiangup](https://github.com/Jixiangup)

## Changed

- `Server` 只负责迁移、Account/ZITADEL 初始化与 Router 组合；连接池、监听器、Axum State、
  日志和关闭策略改由宿主应用管理。— 处理人：[@Jixiangup](https://github.com/Jixiangup)
- 桌面认证入口收敛到 `nexora::desktop`，服务端配置与扩展入口收敛到
  `nexora::server`。— 处理人：[@Jixiangup](https://github.com/Jixiangup)
- `desktop` 与 `server` Cargo feature 分别内置对应 Account 能力，不再要求应用组合内部
  Account feature。— 处理人：[@Jixiangup](https://github.com/Jixiangup)
- 数据库升级完全依据 SQLx 迁移历史执行，删除首次初始化布尔开关。— 处理人：
  [@Jixiangup](https://github.com/Jixiangup)

## Breaking changes

- 删除 `account-client`、`account-server`、`account-zitadel` 等应用侧内部 feature 组合。
- 删除公开路径 `nexora::account::client`。
- 删除 `ServerOptions`、`Server::run` 和 `Server::into_parts`。
- 删除 `initialize_empty_database` 配置。

## 从 0.1.0 升级

1. 桌面依赖只启用 `desktop,derive`，把 `nexora::account::client::*` 改为
   `nexora::desktop::*`。
2. 服务端依赖只启用 `server,derive`，使用 `Server::new()` 和
   `server.initialize(&settings, &pool, setup_secret)`。
3. 应用自行创建 `PgPool`、`TcpListener` 和最终 State，通过
   `Router::new().merge(server.routers())` 合并后调用 `axum::serve(listener, app)`。
4. 把 `[server] bind = "IP:PORT"` 拆成 `ip` 与 `port`，增加 `[setup] secret`、ZITADEL
   `project_id` 和 `personal_access_token`。
5. 删除 `initialize_empty_database`；SQLx 会依据迁移记录自动跳过已经执行的版本。

升级前请备份数据库与生产配置；配置校验失败时先修正字段，不要绕过初始化或迁移检查。
