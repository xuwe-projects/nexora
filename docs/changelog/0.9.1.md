# Nexora 0.9.1

发布日期：2026-07-19

本版本继续处于 `0.x` beta 阶段，不进入 `1.x` 稳定通道；版本号使用标准 SemVer `0.9.1`，
不附加 beta 后缀。本次发布修复主窗口顶部标签栏在 `Segmented` 样式下把置顶标签和普通标签
渲染成两个独立 `TabBar` 时产生的视觉断裂，并修复默认用户管理页激活时可能触发的 GPUI
Entity 重入 panic。

## Fixed

- 顶部 Feature 标签栏改为单个 gpui-component `TabBar` 渲染。置顶标签仍按原规则排在最前面，
  但不再拆成独立置顶标签栏和普通标签栏，因此 `Segmented` 背景、圆角和滑动指示器会保持为
  一整块连续控件。— 处理人：[@Jixiangup](https://github.com/Jixiangup)
- 合并顶部标签栏的滚动状态，使用一个 `ScrollHandle` 跟踪官方 `TabBar` 内置的横向滚动区域；
  点击标签、右键菜单、关闭标签和取消置顶行为保持不变。— 处理人：
  [@Jixiangup](https://github.com/Jixiangup)
- 默认用户管理页在已登录状态下激活并开始加载用户时，不再由子表格反向读取正在更新中的
  `UsersPage`。这修复了点击“用户管理”可能出现的
  `cannot read ... UsersPage while it is already being updated` panic。— 处理人：
  [@Jixiangup](https://github.com/Jixiangup)

## Issue 与 PR

- Issue：无。本版本来自顶部标签栏视觉连续性与默认用户管理点击崩溃修复需求。
- PR：无；按发布要求由默认分支直接提交并发布。

## 兼容性与破坏性变更

- 无破坏性变更。
- 无 HTTP API、数据库迁移、配置项、ZITADEL/Account 契约或脚手架生成结构变更。
- 顶部置顶标签的语义保持为“排在标签列表最前面”；置顶标签不再拥有单独的滚动区域。
- 默认用户管理页只调整内部 GPUI Entity 数据流，不改变用户、角色或权限接口。
- 本版本未修改 `.agents/skills`、宏规则或生成项目约束。下游项目升级时不需要同步 skills。

## 升级

1. 把 Nexora tag 从 `v0.9.0` 更新为 `v0.9.1` 并重新锁定依赖。
2. 如果应用使用默认 `ApplicationTabStyle::Segmented`，升级后顶部标签栏会自动使用连续的单个
   `TabBar` 视觉。
3. 如果应用显式使用 `ApplicationOptions::tab_style(...)` 的其他官方样式，不需要修改代码。
4. 不需要运行数据库迁移、不需要修改服务端配置、不需要同步 `.agents/skills`。
5. 回滚到 `v0.9.0` 只会恢复旧的双 `TabBar` 顶部标签栏布局。
