# Nexora 0.3.1

发布日期：2026-07-18

本补丁版本向桌面宿主公开不含 token 的 Account 认证作用域及其观察器。应用现在可以在退出、
重新登录、替换认证器或切换账号时立即清理自己的会话级 Global，并用作用域 revision 阻止旧
异步响应写回新会话，而不需要访问 Nexora 内部认证状态。

## Added

- 新增 `AccountAuthenticationScope { revision, user_id }`。`user_id` 是当前用户在
  `account.users(id)` 中的本地 ID，未登录时为 `None`；快照不包含 access token、refresh
  token 或 Provider 资料。— 处理人：[@Jixiangup](https://github.com/Jixiangup)
- `nexora::desktop::authentication_scope(&App)` 向宿主提供当前安全作用域快照，可随业务异步
  请求一起捕获并在写回前比较。— 处理人：[@Jixiangup](https://github.com/Jixiangup)
- `nexora::desktop::observe_authentication(&mut Context<T>, callback)` 返回由宿主 Entity 持有
  的 `Subscription`，只在认证作用域实际改变时回调，不会因登录提示文字或忙碌状态变化制造
  无效刷新。— 处理人：[@Jixiangup](https://github.com/Jixiangup)

## Changed

- Account 登录流程现在把认证作用域 revision 与浏览器登录任务 generation 分开维护。成功或
  失败的登录结果提交、退出登录以及认证器替换都会使旧作用域失效；仅开始登录不会过早清除
  当前已认证业务状态。现有 Shell 登录门禁、Feature 与 Sidebar 生命周期保持不变。— 处理人：
  [@Jixiangup](https://github.com/Jixiangup)
- Workspace 组件版本统一同步为 0.3.1，Account 服务端、默认用户与角色管理页面以及数据库
  迁移保持 0.3.0 行为，应用可以用单一 tag 固定本次桌面能力。— 处理人：
  [@Jixiangup](https://github.com/Jixiangup)

## 关联 Issue / PR

- 宿主采用场景：[iMES #220](https://github.com/xuwe-projects/imes/issues/220)。
- PR：无。本版本由默认分支上的直接提交发布，不关联无关 PR。

## 兼容性与破坏性变更

- 本版本只新增公开桌面 API，不删除或改变 0.3.0 的现有 API，属于向后兼容补丁。
- 没有新增、删除或修改数据库 DDL，不需要迁移数据或调整 `_sqlx_migrations`。
- 新快照的 `user_id` 明确是 Nexora 本地 `account.users(id)`，不能替换成 OIDC subject，也不能
  用作跨部署或跨进程标识。
- `revision` 只在当前进程中用于并发隔离，不能持久化、写入数据库或在应用重启后比较。

## 从 0.3.0 升级

1. 把 Nexora 依赖固定到 `v0.3.1` 并更新锁文件：

   ```toml
   nexora = { git = "https://github.com/xuwe-projects/nexora", tag = "v0.3.1", default-features = false }
   ```

   ```bash
   cargo update -p nexora
   ```

2. 需要管理会话级业务 Global 的桌面应用在长生命周期 Entity 中保存观察订阅：

   ```rust
   let initial = nexora::desktop::authentication_scope(cx);
   let subscription = nexora::desktop::observe_authentication(cx, |this, scope, cx| {
       this.authentication_changed(scope, cx);
   });
   ```

3. 每个业务异步请求捕获当前 `AccountAuthenticationScope`；响应返回时与最新快照比较，不一致
   就丢弃结果。业务 HTTP 客户端收到 401 时仍应调用 `nexora::desktop::sign_out(cx)`，让统一
   登录门禁和全部宿主观察器同步清理状态；403 只表示权限不足，不应退出。
4. 重新安装 0.3.1 CLI：

   ```bash
   cargo install --git https://github.com/xuwe-projects/nexora --tag v0.3.1 nexora --locked --force --no-default-features --features cli --bin nexora
   ```

## 回滚注意事项

- 数据库没有变化，不需要 down migration。
- 回退到 0.3.0 前，删除宿主对 `AccountAuthenticationScope`、`authentication_scope` 与
  `observe_authentication` 的调用，并恢复依赖 tag 与锁文件。
- 依赖这些 API 清理业务 Global 的应用不能只回退 Nexora；必须同时回退对应宿主代码，否则
  会因符号缺失而无法编译。
