# Nexora 0.14.0

发布日期：2026-07-21

本版本继续处于 `0.x` beta 阶段，不进入 `1.x` 稳定通道；版本号使用标准 SemVer `0.14.0`。
本次发布为 Account 增加统一头像上传、用户头像更新和 ZITADEL metadata 同步能力。

## Added

- Account contracts 新增 `avatar_url` 创建用户字段、`AvatarUploadResponse` 和
  `UpdateUserAvatarRequest`，HTTP 字段继续使用 `snake_case`。处理人：
  [@zhutao525](https://github.com/zhutao525)
- Account 服务端新增 `AvatarStorage` 抽象、`LocalAvatarStorage` 本地存储实现、头像上传 API
  与用户头像更新 API。上传文件限制为 2 MiB，支持 PNG、JPEG、WebP 和 GIF，返回的 URL 必须是
  HTTPS 或本机 loopback HTTP。处理人：[@zhutao525](https://github.com/zhutao525)
- Account HTTP 新增 `POST /avatars` 和 `PATCH /users/{user_id}/avatar`，并在 OpenAPI
  契约中同步公开。处理人：[@zhutao525](https://github.com/zhutao525)
- ZITADEL 目录适配新增头像 metadata 写入、读取回填和更新能力，metadata key 固定为
  `urn:nexora:account:avatar_url`。处理人：[@zhutao525](https://github.com/zhutao525)

## Changed

- 创建用户时可把头像 URL 同步写入 ZITADEL metadata 与本地账号；读取外部身份时，如果
  ZITADEL profile 没有头像，会回退读取 Nexora metadata。处理人：
  [@zhutao525](https://github.com/zhutao525)
- 更新用户头像时会先同步身份目录，再更新本地数据库；如果本地更新失败，会尽力把身份目录头像
  恢复为原值并记录补偿错误。处理人：[@zhutao525](https://github.com/zhutao525)
- 默认桌面用户开通表单支持填写头像 URL，Account desktop client 暴露头像上传与头像更新方法。
  处理人：[@zhutao525](https://github.com/zhutao525)
- 根 workspace 版本、内部 workspace 依赖约束、CLI 安装示例和 OpenAPI 元数据同步更新到
  `0.14.0`。处理人：[@zhutao525](https://github.com/zhutao525)

## Fixed

- 修正头像 client 新增 rustdoc 的编码问题，并补齐头像更新相关公开 API 的错误条件说明。处理人：
  [@zhutao525](https://github.com/zhutao525)

## 兼容性与破坏性变更

- `ProvisionUserRequest` 新增可选 `avatar_url` 字段，既有 JSON 请求无需修改。
- 新增 `users:avatar.write` 权限。升级后内置 admin 角色会通过迁移自动获得该权限；自定义角色需要
  显式授予后才能上传、更新或清空用户头像。
- 新增迁移 `0009_account_add_user_avatar_permission`，必须运行 Nexora 迁移。回滚到 `v0.13.0`
  时需要回滚该权限及相关角色授权。
- 使用头像上传 API 的宿主必须配置 `AccountDependencies::avatar_storage`；未配置时接口返回服务不可用。
- ZITADEL 头像同步依赖 UserService metadata 能力；Provider 不支持时会返回稳定目录错误，不会静默忽略。

## 升级

1. 把 Nexora tag 从 `v0.13.0` 更新为 `v0.14.0` 并重新锁定依赖。
2. 运行数据库迁移，确保 `users:avatar.write` 权限和 admin 默认授权写入完成。
3. 需要头像上传能力的宿主配置 `AccountDependencies::avatar_storage`，例如使用
   `LocalAvatarStorage::new(directory, public_base_url)`。
4. 给需要管理头像的自定义角色授予 `users:avatar.write`。
5. 客户端先调用 `POST /avatars` 获得 `avatar_url`，再调用 `PATCH /users/{user_id}/avatar`
   写入或清空头像；创建用户时也可直接传入 `avatar_url`。
