# Nexora 0.15.0

发布日期：2026-07-21

本版本继续处于 `0.x` beta 阶段，不进入 `1.x` 稳定通道；版本号使用标准 SemVer `0.15.0`。
本次发布为 iMES 员工创建补齐 ZITADEL human 联系手机号写入能力，并在创建时默认把邮箱与手机号标记为已验证。

## Added

- `CreateHumanIdentity` 新增兼容构造路径 `with_contact_phone(phone)`，可在保持旧结构体字段不变的前提下，把登录手机号同时作为 ZITADEL human phone/mobile 联系信息写入。— 处理人：[@zhutao525](https://github.com/zhutao525)
- `IdentityDirectory` 新增默认兼容方法 `create_human_identity_with_contact`；旧目录适配器会忽略额外手机号，ZITADEL 目录适配器会写入非空手机号。— 处理人：[@zhutao525](https://github.com/zhutao525)
- `ZitadelUserDirectory::create_human_user_with_contact` 与 `ZitadelProvisioningClient::create_human_user_with_contact` 支持在默认 Account 和客户 portal provisioning 两条创建路径中同步已验证联系手机号。— 处理人：[@zhutao525](https://github.com/zhutao525)

## Changed

- ZITADEL UserService v2 `CreateUser` 请求现在使用 `SetHumanEmail.is_verified = true` 创建邮箱，不再发送默认邮箱验证码；传入联系手机号时使用 `SetHumanPhone.is_verified = true`，不发送短信验证码。— 处理人：[@zhutao525](https://github.com/zhutao525)
- 本仓库维护的最小 ZITADEL v2 proto 契约补齐 `SetHumanPhone`、`HumanPhone` 与 `CreateUserRequest.Human.phone` 字段。— 处理人：[@zhutao525](https://github.com/zhutao525)
- 根 workspace 版本、内部 workspace 依赖约束、CLI 安装示例和 OpenAPI 元数据同步更新到 `0.15.0`。— 处理人：[@zhutao525](https://github.com/zhutao525)

## Issue 与 PR

- Issue：无。本版本来自 iMES 创建员工时需要同步 ZITADEL 联系手机号并跳过邮箱/短信二次验证的集成需求。
- PR：无；按发布要求由当前分支直接提交并发布。

## 兼容性与破坏性变更

- 无破坏性变更。旧的 `CreateHumanIdentity { ... }` 结构体字面量和 `Account::create_managed_user_with_roles(request, ...)` 调用继续可用。
- 未提供或传入空白联系手机号时，ZITADEL 创建请求不写入 phone 字段，保持旧行为。
- 本版本不新增数据库迁移，Nexora Account 本地库仍只保存身份资料快照，不新增通用 phone 字段。

## 升级

1. 把 Nexora tag 更新为 `v0.15.0` 并重新锁定依赖。
2. iMES 创建员工时继续把手机号放入 `CreateHumanIdentity.username`，并额外调用 `.with_contact_phone(phone)` 后传给 `Account::create_managed_user_with_roles`。
3. 电子邮箱继续放入 `CreateHumanIdentity.email`；ZITADEL 默认实现会以已验证状态创建 email 和 contact phone，不要求员工再收验证码。

示例：

```rust
let request = CreateHumanIdentity {
    username: phone.clone(),
    given_name,
    family_name,
    email,
    display_name,
    initial_password,
    require_password_change: false,
    avatar_url: None,
}
.with_contact_phone(phone);

account
    .create_managed_user_with_roles(request, role_ids, granted_by)
    .await?;
```
