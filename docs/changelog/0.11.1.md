# Nexora 0.11.1

发布日期：2026-07-19

本版本继续处于 `0.x` beta 阶段，不进入 `1.x` 稳定通道；版本号使用标准 SemVer `0.11.1`，
不附加 beta 后缀。本次只修复和收敛默认桌面管理体验，没有公开 Rust API、HTTP 契约、配置、
数据库迁移或脚手架规则的不兼容变更，因此在 `0.11.x` 内升级 patch 版本。

## Fixed

- 默认“角色与权限”创建表单不再向操作者展示“角色键”输入；默认客户端会根据角色名称生成隐藏
  `CreateRoleRequest.key`，并在已加载角色 key 内做后缀避让，服务端继续用既有格式校验和唯一
  约束兜底。— 处理人：[@Jixiangup](https://github.com/Jixiangup)
- 默认角色编辑表单不再展示只读“角色键”字段，避免把授权标识当作可编辑业务表单信息干扰操作者。
  角色列表仍保留 key 作为只读识别信息。— 处理人：[@Jixiangup](https://github.com/Jixiangup)
- `PanelDialog` 的内容区改为在对话框 surface 内独立纵向滚动，`FormDialog` 默认限制为当前
  Feature Panel 高度的 80%，并保留响应式宽度上限；字段过多时标题和 footer 不再被长表单挤出
  panel。— 处理人：[@Jixiangup](https://github.com/Jixiangup)

## Issue 与 PR

- Issue：无。本版本来自默认 Account 角色管理表单和 FormDialog 响应式布局修复。
- PR：无；按发布要求由默认分支直接提交并发布。

## 兼容性与破坏性变更

- 无公开 API 破坏性变更。`CreateRoleRequest.key` 契约保持不变；变化只发生在 Nexora 默认桌面
  角色创建页面，用户不再手动输入该字段。
- 无数据库迁移、OpenAPI 路由、服务端配置或生成项目结构变更。
- 回滚到 `v0.11.0` 只会恢复旧的默认角色表单和旧的 FormDialog 高度行为；不会涉及数据迁移。

## 升级

1. 把 Nexora tag 从 `v0.11.0` 更新为 `v0.11.1` 并重新锁定依赖。
2. 使用默认 Account 角色管理页的桌面应用无需改代码；创建角色时继续填写角色名称、说明和权限即可。
3. 直接使用 `nexora::desktop::FormDialog` 的应用会自动获得新的默认响应式高度和内容区滚动行为。
4. 不需要同步新的脚手架 Skill、宏规则或数据库迁移；本版本未修改生成项目约束。
