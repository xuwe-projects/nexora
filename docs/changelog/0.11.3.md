# Nexora 0.11.3

发布日期：2026-07-20

本版本继续处于 `0.x` beta 阶段，不进入 `1.x` 稳定通道；版本号使用标准 SemVer `0.11.3`，
不附加 beta 后缀。本次是 `0.11.2` 的补丁修正，只约束 SQL 文件换行符、同步发布版本元数据
和安装文档，没有公开 Rust API、HTTP 契约、配置、数据库迁移内容或脚手架规则的不兼容变更。

## Fixed

- 仓库新增 `.gitattributes`，强制 `*.sql` 文件使用 LF 换行，避免数据库迁移文件在 Windows
  checkout 时被转换为 CRLF，导致跨平台迁移字节内容不一致。—
  处理人：[@Jixiangup](https://github.com/Jixiangup)

## Changed

- 根 workspace 版本、内部 workspace 依赖约束、CLI 安装示例和 OpenAPI 元数据同步更新到
  `0.11.3`。—
  处理人：[@Jixiangup](https://github.com/Jixiangup)

## Issue 与 PR

- Issue：无。本版本来自发布前对框架迁移文件跨平台换行符稳定性的修正。
- PR：无；按发布要求由默认分支直接提交并发布。

## 兼容性与破坏性变更

- 无公开 API 破坏性变更。
- 无数据库表结构、迁移 SQL 语义、OpenAPI 路由、服务端配置或生成项目结构变更。
- 已存在的迁移文件内容保持不变；本版本只让 Git 在不同操作系统上统一以 LF 签出 SQL 文件。
- 回滚到 `v0.11.2` 会移除该换行符约束，不涉及数据迁移。

## 升级

1. 把 Nexora tag 从 `v0.11.2` 更新为 `v0.11.3` 并重新锁定依赖。
2. 使用框架内置迁移或生成项目的应用无需修改业务代码。
3. 不需要执行手工数据库迁移；迁移文件语义与 `0.11.2` 一致。
4. 不需要同步新的脚手架 Skill、宏规则或生成项目约束；本版本未修改这些规则。
