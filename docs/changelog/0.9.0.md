# Nexora 0.9.0

发布日期：2026-07-19

本版本继续处于 `0.x` beta 阶段，不进入 `1.x` 稳定通道；在 GPUI 与 gpui-component 未发布 crates
之前，Nexora 仍以 `0.x` 版本迭代。本次发布把标准 CRUD Panel 进一步沉淀为可复用的
`CrudTableRow`/`CrudTableDelegate` 表格增强能力，补齐 Account 默认管理页面细节，并开放顶部
标签栏样式 API。

## Added

- 新增 `nexora::desktop::{CrudTableRow, CrudTableDelegate}`，用于把普通行数据结构接入
  gpui-component `DataTable`。`CrudTableRow` 可通过 `#[derive(nexora::CrudTableRow)]` 生成
  列定义、表头/正文对齐、单元格渲染和文本导出；操作列通过 delegate 的 `action_column`
  追加，复杂场景仍可直接手写原生 `TableDelegate`。— 处理人：
  [@Jixiangup](https://github.com/Jixiangup)
- 新增 `nexora::desktop::TableCell`，作为 CRUD DataTable 正文单元格的默认增强组件；正文
  默认垂直居中、水平靠左，并提供水平和垂直对齐 API。— 处理人：
  [@Jixiangup](https://github.com/Jixiangup)
- 新增 `nexora::ApplicationTabStyle` 与 `ApplicationOptions::tab_style`，顶部 Feature 标签默认
  使用 gpui-component 官方 `TabBar::segmented()`，并允许应用切换到 `Underline`、`Pill` 或
  `Outline`。— 处理人：[@Jixiangup](https://github.com/Jixiangup)
- `UserResponse` 新增 `user_type` 字段，区分 `human` 与 `service_account`。默认 Account 管理
  页面会展示服务账号类型，并禁止对服务账号执行状态或角色操作。— 处理人：
  [@Jixiangup](https://github.com/Jixiangup)
- 新增 `crates/macros/MACRO_AUDIT.md`，记录现有派生宏的 `cargo expand` 展开、手写等价对比、
  `cargo bench` 与 `cargo bloat` 审查证据。— 处理人：
  [@Jixiangup](https://github.com/Jixiangup)

## Changed

- 默认用户管理表格迁移到 `CrudTableRow` + `CrudTableDelegate`，补齐用户类型列、正文垂直居中、
  操作列居中和无限滚动加载边界。— 处理人：[@Jixiangup](https://github.com/Jixiangup)
- 默认用户管理与角色权限的创建弹窗文案改为业务语义，不再在应用内向用户暴露 ZITADEL 等实现
  细节。— 处理人：[@Jixiangup](https://github.com/Jixiangup)
- 默认角色与权限页面继续使用标准 `CrudPanel`，刷新按钮图标和用户管理保持一致。— 处理人：
  [@Jixiangup](https://github.com/Jixiangup)
- 生成项目的 `.agents/skills` 已同步 CRUD Panel、CRUD DataTable、顶部标签样式和自定义宏验证
  规则；符合标准 CRUD Panel 的业务页面应优先使用新组件。— 处理人：
  [@Jixiangup](https://github.com/Jixiangup)

## Fixed

- 修复 `FormDialog` 内容区域因布局约束导致创建用户、创建角色弹窗主体无法展示的问题。—
  处理人：[@Jixiangup](https://github.com/Jixiangup)
- 修复默认用户表格状态列、操作列缺少表头分隔与内容对齐不一致的问题；表格表头默认水平、
  垂直居中，正文默认垂直居中、水平靠左，并允许按列覆盖。— 处理人：
  [@Jixiangup](https://github.com/Jixiangup)
- 修复服务账号仍可在默认管理界面触发状态或角色操作的问题；服务端也拒绝修改服务账号状态与
  直接角色。— 处理人：[@Jixiangup](https://github.com/Jixiangup)

## Issue 与 PR

- Issue：无。本版本来自默认 Account 管理体验、标准 CRUD Panel 抽象与宏规则升级需求。
- PR：无；按发布要求由默认分支直接提交并发布。

## 兼容性与破坏性变更

- 破坏性变更：`UserResponse` JSON 响应新增必填 `user_type` 字段；严格反序列化客户端需要更新
  DTO。
- 破坏性变更：标准 CRUD 表格推荐路径从手写 `TableDelegate` 样板迁移到
  `CrudTableRow`/`CrudTableDelegate`；旧页面仍可手写原生 `TableDelegate`，但新脚手架与 skill
  会优先生成新 API。
- 破坏性变更：服务账号不可再通过默认 Account API 修改状态或直接角色，相关请求返回
  `409 service_account_immutable`。
- 顶部标签默认视觉变为 gpui-component 官方 `Segmented`；应用可通过
  `ApplicationOptions::tab_style` 切换视觉样式。
- 数据库 schema、迁移历史、`gpui` revision 与 `gpui-component` revision 均未变化。
- 本版本仍是 `0.x` beta。GPUI 与 gpui-component 正式发布 crates 之前，Nexora 不进入 `1.x`
  稳定发布线。

## 升级

1. 把 Nexora tag 从 `v0.8.0` 更新为 `v0.9.0` 并重新锁定依赖。
2. 同步新脚手架中的 `.agents/skills`，尤其是标准 CRUD Panel、`CrudTableRow`、顶部标签样式和
   自定义宏验证闭环规则。
3. 如果客户端严格解析 `User`/`UserResponse`，新增 `user_type: "human" | "service_account"`。
4. 标准资源管理页建议改用 `nexora::desktop::{CrudPanel, CrudPanelToolbar}`；CRUD 表格建议使用
   `#[derive(nexora::CrudTableRow)]` + `CrudTableDelegate<T>`，操作列通过 `action_column` 追加。
5. 如果应用希望保留旧标签视觉，通过
   `ApplicationOptions::tab_style(ApplicationTabStyle::Underline | Pill | Outline)` 显式选择。
6. 升级后建议在本地执行一次 `cargo clean`，再重新 `cargo check` 或启动应用，避免旧 GPUI
   构建产物干扰排查。
7. 不需要运行数据库迁移或修改服务端配置。回滚到 `v0.8.0` 会移除新增表格增强 API 和
   `user_type` 响应字段。
