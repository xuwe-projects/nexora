# Nexora 0.4.1

发布日期：2026-07-18

本补丁版本修复 0.4.0 文档站中机器可读 OpenAPI 契约未被 VitePress 发布的问题。
`v0.4.0` 标签与 Release 保持不可变；修复按严格 Semantic Versioning 递增 PATCH 到 0.4.1。

## Fixed

- 将唯一维护的 OpenAPI 3.1 源文件放入 VitePress `public` 目录，使构建产物稳定包含
  `/openapi.yaml`，中英文 HTTP/Rust 服务端参考中的机器可读契约链接不再返回 404。—
  处理人：[@Jixiangup](https://github.com/Jixiangup)
- 配置说明同步指向新的仓库源文件路径 `docs/public/openapi.yaml`。— 处理人：
  [@Jixiangup](https://github.com/Jixiangup)

## 关联 Issue / PR

- Issue：无。该问题由 0.4.0 发布后的 Pages 可访问性验证发现。
- PR：无。本版本不编造或关联未实际承载改动的 PR。

## 兼容性

- Rust、HTTP、数据库和配置契约均未变化。
- OpenAPI 接口定义与 0.4.0 相同；仅 `info.version`、静态源文件位置和 Pages 发布结果变化。
- 该向后兼容文档发布修复严格递增 PATCH；无需数据库迁移。

## 从 0.4.0 升级

1. 把 Nexora 依赖固定到 `v0.4.1` 并更新锁文件：

   ```toml
   nexora = { git = "https://github.com/xuwe-projects/nexora", tag = "v0.4.1", default-features = false }
   ```

2. 不需要新增迁移或代码调整。
3. 使用 `https://xuwe-projects.github.io/nexora/openapi.yaml` 获取机器可读契约。

## 回滚注意事项

- 回退到 0.4.0 不涉及数据库或运行时代码变化，但 Pages 上的 `/openapi.yaml` 会再次不可用。
- 不要移动或重写已经发布的 `v0.4.0` 与 `v0.4.1` 标签。
