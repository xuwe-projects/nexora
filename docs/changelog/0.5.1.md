# Nexora 0.5.1

发布日期：2026-07-18

本补丁版本修复 `FormDialog::submit_disabled` 的按钮禁用映射。公开 API、配置、数据库迁移和
依赖 revision 均保持与 0.5.0 兼容，因此严格按 Semantic Versioning 递增 PATCH。

## Fixed

- 修正 FormDialog footer：取消按钮仅在表单正在提交时禁用；提交按钮在正在提交或调用方设置
  `submit_disabled(true)` 时禁用。业务校验未通过时，用户现在仍能取消或处理未保存草稿，
  同时无法误触提交。— 处理人：[@Jixiangup](https://github.com/Jixiangup)
- 增加 GPUI 真实点击回归测试和源码契约测试，分别验证按钮交互结果与禁用条件不会再次对调。
  — 处理人：[@Jixiangup](https://github.com/Jixiangup)

## Issue 与 PR

- Issue：无。本版本修复来自 iMES 接入 v0.5.0 公共 FormDialog 时发现的可复现问题。
- PR：无；由默认分支直接修复并发布。

## 兼容性

- 无破坏性变更，无需修改 FormDialog 调用代码。
- `submit_disabled(true)` 的行为恢复为 0.5.0 rustdoc 已声明的契约：只控制提交按钮。
- GPUI、gpui-component、ZITADEL、HTTP API 和数据库 schema 均未变化。

## 升级

把 Nexora tag 从 `v0.5.0` 更新为 `v0.5.1` 并重新锁定依赖即可。已发布的 `v0.5.0` tag 保持
不变，不需要运行数据库迁移或更改配置。回滚只需恢复 `v0.5.0` 依赖，但会重新引入按钮禁用
映射错误，因此不建议回滚。
