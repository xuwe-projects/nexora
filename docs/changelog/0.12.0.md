# Nexora 0.12.0

发布日期：2026-07-20

本版本继续处于 `0.x` beta 阶段，不进入 `1.x` 稳定通道；版本号使用标准 SemVer `0.12.0`。
本次发布为 Nexora Account 增加创建人类用户时设置初始密码的能力，并按用户要求采用直接破坏现有公开
API 的最简洁实现，因此从 `0.11.3` 升级时需要同步调整调用方请求体和 Rust 结构体初始化代码。

## Added

- Account 创建人类用户时支持携带 `initial_password` 与 `require_password_change`，ZITADEL UserService
  v2 创建路径会把初始密码写入外部身份目录，同时继续只在 Nexora 本地数据库保存用户绑定和角色授权。
  密码不会写入本地明文字段、日志或错误详情。— 处理人：[@Jixiangup](https://github.com/Jixiangup)
- HTTP/client 契约的 `ProvisionUserRequest` 新增 `initial_password` 必填字段和
  `require_password_change` 可选字段，供 iMES 创建员工时用手机号作为账号名并传入
  `imes{手机号}.` 默认初始密码。— 处理人：[@Jixiangup](https://github.com/Jixiangup)

## Changed

- `CreateHumanIdentity` 现在直接包含 `initial_password` 与 `require_password_change`，宿主应用通过
  `nexora::server::Account::create_managed_user_with_roles` 创建托管用户时必须提供初始密码。—
  处理人：[@Jixiangup](https://github.com/Jixiangup)
- 默认 Account 用户创建界面会为没有密码输入框的人工创建流程生成一次性初始密码，并要求用户首次登录后修改。
  业务系统仍应优先通过后端 facade 显式传入自己的初始密码规则。— 处理人：[@Jixiangup](https://github.com/Jixiangup)
- 根 workspace 版本、内部 workspace 依赖约束、CLI 安装示例和 OpenAPI 元数据同步更新到 `0.12.0`。—
  处理人：[@Jixiangup](https://github.com/Jixiangup)

## Issue 与 PR

- Issue：无。本版本来自 iMES 员工创建流程对 Nexora Account 初始密码写入能力的直接集成需求。
- PR：无；按发布要求由默认分支直接提交并发布。

## 兼容性与破坏性变更

- 存在公开 API 破坏性变更：`CreateHumanIdentity` 和 `ProvisionUserRequest` 现在需要
  `initial_password`，旧的无密码创建请求不能再编译或通过服务端校验。
- 旧 Rust 写法：

  ```rust
  CreateHumanIdentity {
      username,
      given_name,
      family_name,
      email,
      display_name,
  }
  ```

- 新 Rust 写法：

  ```rust
  CreateHumanIdentity {
      username,
      given_name,
      family_name,
      email,
      display_name,
      initial_password,
      require_password_change: false,
  }
  ```

- 旧 HTTP 请求体缺少 `initial_password`；新请求体必须增加：

  ```json
  {
    "initial_password": "imes13800000000.",
    "require_password_change": false
  }
  ```

- 无数据库 schema、迁移 SQL、服务端配置或脚手架规则变更；回滚到 `v0.11.3` 时需要同步撤回调用方新增的
  初始密码字段。

## 升级

1. 把 Nexora tag 从 `v0.11.3` 更新为 `v0.12.0` 并重新锁定依赖。
2. 所有调用 `CreateHumanIdentity` 或 `ProvisionUserRequest` 的业务模块必须补充 `initial_password`。
3. iMES 员工创建流程应使用手机号作为 `username`，并把桌面端提交或默认生成的 `imes{手机号}.` 作为
   `initial_password` 传给 Nexora Account；`require_password_change` 按业务策略设置。
4. 不需要执行手工数据库迁移，也不需要同步新的脚手架 Skill、宏规则或生成项目约束。
