# Nexora 0.10.0

发布日期：2026-07-19

本版本继续处于 `0.x` beta 阶段，不进入 `1.x` 稳定通道；版本号使用标准 SemVer `0.10.0`，
不附加 beta 后缀。由于本次包含 `FormDialog` 公开 API 调整和默认标签视觉变更，在 `0.x`
阶段按 SemVer 升级 minor 版本。

## Added

- 新增 `nexora::desktop::{FormItem, FormItemControl}`，为标准创建/编辑表单提供标签、说明、
  必填标记以及 `input`、`password_input`、`number_input`、`checkbox` 等常用控件封装。—
  处理人：[@Jixiangup](https://github.com/Jixiangup)
- 默认设置窗口新增“默认显示器”选项，支持选择主窗口下次启动时打开到哪个显示器；显示器未连接
  时会回退到系统主显示器。— 处理人：[@Jixiangup](https://github.com/Jixiangup)
- 默认用户管理页和角色与权限页新增标准 CRUD Panel 工具栏筛选/查询能力，支持关键词、状态、
  类型或角色类型过滤。— 处理人：[@Jixiangup](https://github.com/Jixiangup)

## Changed

- `FormDialog` 改为 builder API：使用
  `FormDialog::new(id, state).title(...).child(FormItem::new(...)).section(...)` 组合内容，并通过
  `.with_size(theme::component_size(cx))` 跟随设置中的组件尺寸。— 处理人：
  [@Jixiangup](https://github.com/Jixiangup)
- 主窗口顶部 Feature 标签默认样式从 `Segmented` 改为 gpui-component 官方 story 中的默认
  `Tabs` 样式；`ApplicationTabStyle` 现在支持 `Tab`、`Underline`、`Pill`、`Outline` 与
  `Segmented`。— 处理人：[@Jixiangup](https://github.com/Jixiangup)
- `CrudPanel`、默认 Account 页面按钮、筛选输入、下拉、角色编辑和用户创建表单统一使用
  `theme::component_size(cx)`，跟随设置里的组件尺寸。— 处理人：
  [@Jixiangup](https://github.com/Jixiangup)
- 默认创建用户、创建角色和角色管理弹窗只展示业务文案，不再让最终用户关注底层身份提供方细节；
  服务账号在用户表格中展示类型，并保持不可操作。— 处理人：
  [@Jixiangup](https://github.com/Jixiangup)
- 生成项目 `.agents/skills` 与模板同步更新：标准 CRUD Panel、FormDialog、官方 Tab 样式和组件
  尺寸跟随设置成为默认开发约束。— 处理人：[@Jixiangup](https://github.com/Jixiangup)

## Fixed

- 修复 `PanelDialog` 内容区布局导致 FormDialog 正文可见但底部按钮落在 surface 命中范围外的问题；
  创建用户、创建角色等弹窗现在会正确展示内容并可点击取消/提交。— 处理人：
  [@Jixiangup](https://github.com/Jixiangup)
- 修复默认用户管理页与角色权限页中刷新、查询、创建按钮尺寸不一致的问题。— 处理人：
  [@Jixiangup](https://github.com/Jixiangup)

## 兼容性与破坏性变更

- 破坏性变更：旧的 `FormDialog::new(id, state, title, content, on_submit)` 写法已移除。请迁移到
  builder API：

```rust
FormDialog::new("create-user-dialog", self.form.clone())
    .title("创建用户")
    .description("填写信息后创建用户。")
    .child(FormItem::new("姓名").required().input(&self.name))
    .section(permission_list)
    .submit_label("创建用户")
    .submit_disabled(!self.can_submit(cx))
    .with_size(theme::component_size(cx))
    .on_submit(cx.listener(Self::submit))
```

- 破坏性行为变更：`PanelDialog` 默认不再通过点击遮罩关闭；表单类弹窗应通过取消、右上角关闭或
  提交表达明确意图。需要恢复遮罩关闭时显式调用 `.overlay_closable(true)`。
- 行为变更：未显式设置 `ApplicationOptions::tab_style(...)` 的应用会从 `Segmented` 视觉切换到
  官方默认 `Tabs` 视觉。需要保持旧视觉时显式设置
  `ApplicationTabStyle::Segmented`。
- 无 HTTP API、数据库迁移、服务端配置或 Account/ZITADEL 接口契约变更。

## 升级

1. 把 Nexora tag 从 `v0.9.1` 更新为 `v0.10.0` 并重新锁定依赖。
2. 全量搜索 `FormDialog::new(`，把旧构造函数迁移为新的 builder API，并引入
   `FormItem` / `FormItemControl`。
3. 如果应用希望继续使用旧的分段标签视觉，在 `ApplicationOptions` 中显式设置
   `ApplicationTabStyle::Segmented`；否则使用新的官方默认 `Tabs` 样式。
4. 同步新脚手架中的 `.agents/skills`，尤其是标准 CRUD Panel、FormDialog builder、组件尺寸
   跟随设置和官方 Tab 样式规则。符合标准 CRUD Panel 的业务页面应优先使用
   `nexora::desktop::{CrudPanel, CrudPanelToolbar}`。
5. 新增/维护自定义宏时继续接入宏验证闭环：`cargo expand` 展开审查、与手写等价代码对比、
   `cargo bench`、`cargo bloat`，未达标时用 `cargo asm` 定位后再迭代。
6. 不需要运行数据库迁移或修改服务端配置。回滚到 `v0.9.1` 需要同步回退 `FormDialog` 调用代码。
