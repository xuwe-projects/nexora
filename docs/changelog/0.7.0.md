# Nexora 0.7.0

发布日期：2026-07-18

本 early-alpha 版本为桌面应用增加可组合的 GPUI 资产注册入口。应用可以把自己的
`assets/icons/**/*.svg` 编译进最终程序，并继续通过 Nexora 现有的 Feature 与
NavigationGroup `icon = "..."` 约定使用这些图标。该能力是向后兼容的新功能，因此在
`0.y.z` 阶段按 Semantic Versioning 递增 MINOR。

## Added

- 新增 `desktop::ApplicationAssets` 与 `desktop::Application::with_asset_source`，底层运行器
  会优先查询应用资产，再回退到 Nexora 内置图标与 `gpui-component-assets` 默认资源。— 处理人：
  [@Jixiangup](https://github.com/Jixiangup)
- 新增 `nexora::ApplicationOptions::application_assets(...)`，应用可以在进入 GPUI 事件循环前
  注册自定义 `AssetSource`，无需绕过 Nexora 的标准启动流程。— 处理人：
  [@Jixiangup](https://github.com/Jixiangup)
- `nexora create` / `nexora init` 生成的桌面入口现在默认声明 `AppAssets`，使用 `rust_embed`
  嵌入 `assets/icons/**/*.svg`，并创建 `assets/icons` 目录。— 处理人：
  [@Jixiangup](https://github.com/Jixiangup)

## Changed

- 脚手架 README 增加自定义图标约定说明：`icon = "warehouse"` 会读取
  `assets/icons/warehouse.svg`，子目录路径例如 `icon = "app/warehouse"` 对应
  `assets/icons/app/warehouse.svg`。— 处理人：
  [@Jixiangup](https://github.com/Jixiangup)
- 更新脚手架生成测试与应用选项测试，锁定应用资产源保存、模板渲染和 `assets/icons` 目录生成
  行为。— 处理人：[@Jixiangup](https://github.com/Jixiangup)

## Issue 与 PR

- Issue：无。本版本来自桌面应用自定义图标接入需求。
- PR：无；按发布要求由默认分支直接提交并发布。

## 兼容性与破坏性变更

- 破坏性变更：无。
- 未配置 `application_assets` 的已有应用行为不变，仍只使用 Nexora 与 `gpui-component-assets`
  的默认资源。
- 新生成项目会增加 `rust-embed` 依赖，并默认包含可为空的 `assets/icons` 目录；该目录为空时
  `AppAssets` 仍能编译通过。
- HTTP API、OpenAPI 路径、ZITADEL 集成、数据库 schema、迁移历史、`gpui` revision 与
  `gpui-component` revision 均未变化。

## 升级

1. 把 Nexora tag 从 `v0.6.0` 更新为 `v0.7.0` 并重新锁定依赖。
2. 需要使用自定义图标的桌面应用，在自己的应用 crate 中实现 `AssetSource`，或参考新脚手架生成的
   `AppAssets` 把 SVG 放入 `assets/icons/**/*.svg`。
3. 在应用选项中调用 `.application_assets(AppAssets)`；已有项目不需要自定义图标时无需修改。
4. 不需要运行数据库迁移或修改服务端配置。回滚到 `v0.6.0` 会移除框架级应用资产注册入口，但不会
   影响未使用该入口的应用。
