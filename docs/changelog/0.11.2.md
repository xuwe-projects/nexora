# Nexora 0.11.2

发布日期：2026-07-19

本版本继续处于 `0.x` beta 阶段，不进入 `1.x` 稳定通道；版本号使用标准 SemVer `0.11.2`，
不附加 beta 后缀。本次是 `0.11.1` 的补丁修正，只调整桌面端 `FormDialog` 的默认高度语义和
发布说明，没有公开 Rust API、HTTP 契约、配置、数据库迁移或脚手架规则的不兼容变更。

## Fixed

- `FormDialog` 常规创建/编辑表单的 surface 现在默认固定为当前 Feature Panel 高度的 80%，
  而不是只设置最大高度；标题和底部操作区保持固定，表单内容区继续独立纵向滚动。—
  处理人：[@Jixiangup](https://github.com/Jixiangup)
- `PanelDialog` overlay 不再用内边距缩小百分比高度的计算基准，`relative(0.8)` 现在直接以
  Panel 本身为参照，避免对话框比 Panel 高度的 80% 少一截。—
  处理人：[@Jixiangup](https://github.com/Jixiangup)

## Added

- `FormDialog::panel_height_ratio(ratio)` 允许应用调整常规表单相对 Panel 的高度比例，默认值为
  `0.8`。
- `FormDialog::auto_height()` 允许极少字段的小表单恢复内容自适应高度，同时仍保留默认 80%
  Panel 高度上限，字段过多时继续由内容区滚动。

## Issue 与 PR

- Issue：无。本版本来自 `0.11.1` 发布后对 `FormDialog` Panel 高度语义的修正。
- PR：无；按发布要求由默认分支直接提交并发布。

## 兼容性与破坏性变更

- 无公开 API 破坏性变更。默认 `FormDialog` 视觉高度会变为当前 Panel 的 80%；这是对
  `0.11.1` 错误默认行为的修复。需要旧的内容自适应高度时可显式调用
  `FormDialog::auto_height()`。
- 无数据库迁移、OpenAPI 路由、服务端配置或生成项目结构变更。
- 回滚到 `v0.11.1` 会恢复只设置最大高度的行为，不涉及数据迁移。

## 升级

1. 把 Nexora tag 从 `v0.11.1` 更新为 `v0.11.2` 并重新锁定依赖。
2. 使用 `nexora::desktop::FormDialog` 的应用无需改代码即可获得正确的 Panel 80% 默认高度。
3. 只有确实想保留小表单内容自适应高度的页面，才需要在构造表单时追加
   `FormDialog::auto_height()`。
4. 不需要同步新的脚手架 Skill、宏规则或数据库迁移；本版本未修改生成项目约束。
