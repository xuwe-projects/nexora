# Nexora 0.16.0

发布日期：2026-07-24

本版本继续处于 `0.x` beta 阶段，不进入 `1.x` 稳定通道；版本号使用标准 SemVer `0.16.0`。
本次发布为 iMES 客户门户复用 Nexora Account/RBAC 提供角色 owner 作用域，避免业务库复制角色与权限表。

## Added

- Account 角色新增 `owner` 字段并通过 append-only 迁移 `0010_account_role_owner_scope` 回填默认值 `IMES`，用于隔离后台系统角色和客户门户角色。— 处理人：[@zhutao525](https://github.com/zhutao525)
- 预置全局门户管理员系统角色 `portal_admin`，`owner = "IMES"`，`is_system = true`；权限集合由宿主启动时同步，普通角色编辑和删除入口不可修改该角色。— 处理人：[@zhutao525](https://github.com/zhutao525)
- 新增 owner 作用域可信宿主 API：`create_role_for_owner`、`create_generated_role_for_owner`、`replace_role_permissions_for_owner`、`replace_user_roles_for_owner`、`ensure_system_role_with_permissions` 和 `grant_user_role`，并从 `nexora::server` / `nexora::account` 公开导出。— 处理人：[@zhutao525](https://github.com/zhutao525)
- HTTP/OpenAPI 契约新增 `owner` 字段和默认 `IMES` 的 owner 查询/写入语义，旧请求缺省 owner 时继续按后台系统范围处理。— 处理人：[@zhutao525](https://github.com/zhutao525)

## Changed

- 默认 `roles`、`role`、`create_role`、`replace_role_permissions` 和 `replace_user_roles` 语义继续面向 `owner = "IMES"`，保持既有后台用户角色管理兼容。— 处理人：[@zhutao525](https://github.com/zhutao525)
- `replace_user_roles_for_owner` 只替换指定 owner 下的用户角色，不影响 `IMES` 或其他客户 owner；`grant_user_role` 幂等追加单个角色且不会清空已有角色。— 处理人：[@zhutao525](https://github.com/zhutao525)
- 角色 key 继续保持全局唯一；`create_generated_role_for_owner` 使用数据库序列生成 `role_<id>` 形式的全局唯一 key，适合门户角色无需人工填写编码的场景。— 处理人：[@zhutao525](https://github.com/zhutao525)
- 根 workspace 版本、内部 workspace 依赖约束、CLI 安装示例和 OpenAPI 元数据同步更新到 `0.16.0`。— 处理人：[@zhutao525](https://github.com/zhutao525)

## Issue 与 PR

- Issue：无。本版本来自 iMES 客户门户接入 Account/RBAC 的框架能力需求。
- PR：无；按发布要求由当前分支直接提交并发布。

## 兼容性与破坏性变更

- 无破坏性变更。旧客户端不传 `owner` 的角色创建、角色权限替换和用户角色替换请求会默认进入 `IMES` 后台范围。
- `Role` / `RoleResponse` 响应新增 `owner` 字段；这是向后兼容的响应字段增加。
- 数据库迁移新增 `account.roles.owner TEXT NOT NULL DEFAULT 'IMES'`，并保留全局唯一 `roles.key` 约束。

## 升级

1. 把 Nexora tag 更新为 `v0.16.0` 并重新锁定依赖。
2. 运行 Nexora 合并后的 SQLx 迁移，确保 `0010_account_role_owner_scope` 已应用。
3. iMES 客户门户角色使用客户 ID 字符串作为 owner；后台系统角色和 `portal_admin` 继续使用 `IMES`。
4. 宿主启动时先注册门户权限，再调用 `ensure_system_role_with_permissions(PORTAL_ADMIN_ROLE_KEY, ...)` 同步全局门户管理员角色权限。
5. 需要给门户管理员默认绑定系统角色时调用 `grant_user_role`；需要替换客户范围角色时调用 `replace_user_roles_for_owner`。
