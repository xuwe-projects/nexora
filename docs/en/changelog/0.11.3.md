# Nexora 0.11.3

Release date: 2026-07-20

This release remains in the `0.x` beta line and does not move Nexora to `1.x`. The version uses the
standard SemVer value `0.11.3` without a beta suffix. This is a patch correction for `0.11.2` and
only constrains SQL file line endings, updates release metadata, and refreshes installation docs. It
does not introduce incompatible public Rust API, HTTP contract, configuration,
database-migration-content, or scaffold-rule changes.

## Fixed

- Added `.gitattributes` so `*.sql` files are always checked out with LF line endings. This prevents
  database migration files from being converted to CRLF on Windows and keeps migration bytes stable
  across platforms. — Owner: [@Jixiangup](https://github.com/Jixiangup)

## Changed

- Updated the root workspace version, internal workspace dependency constraints, CLI installation
  examples, and OpenAPI metadata to `0.11.3`. — Owner:
  [@Jixiangup](https://github.com/Jixiangup)

## Issues and PRs

- Issue: none. This release comes from a pre-release correction to database-migration line-ending
  stability across platforms.
- PR: none; committed and released directly from the default branch as requested.

## Compatibility and Breaking Changes

- No public API breaking changes.
- No database schema, migration SQL semantics, OpenAPI route, server configuration, or generated
  project structure changes.
- Existing migration file contents are unchanged; this release only makes Git check out SQL files
  with LF on every operating system.
- Rolling back to `v0.11.2` removes this line-ending constraint. No data migration is involved.

## Upgrade

1. Update the Nexora tag from `v0.11.2` to `v0.11.3` and refresh the lockfile.
2. Apps using the framework migrations or generated projects do not need business-code changes.
3. No manual database migration is required; migration semantics are identical to `0.11.2`.
4. No generated-project Skill sync, macro-rule update, or generated-project constraint update is
   required. This release does not change those rules.
