# Nexora 0.7.0

Release date: 2026-07-18

This early-alpha release adds a composable GPUI asset registration entry point for desktop
applications. Applications can compile their own `assets/icons/**/*.svg` files into the final
binary and keep using Nexora's existing Feature and NavigationGroup `icon = "..."` convention. This
is a backward-compatible feature, so the release increments MINOR under Semantic Versioning for the
`0.y.z` line.

## Added

- Added `desktop::ApplicationAssets` and `desktop::Application::with_asset_source`. The desktop
  runner now checks application assets first, then falls back to Nexora built-in icons and
  `gpui-component-assets`. — Owner: [@Jixiangup](https://github.com/Jixiangup)
- Added `nexora::ApplicationOptions::application_assets(...)`, allowing applications to register a
  custom `AssetSource` before the GPUI event loop starts without bypassing Nexora's standard
  startup flow. — Owner: [@Jixiangup](https://github.com/Jixiangup)
- `nexora create` / `nexora init` desktop entries now declare `AppAssets` by default, embed
  `assets/icons/**/*.svg` through `rust_embed`, and create the `assets/icons` directory. — Owner:
  [@Jixiangup](https://github.com/Jixiangup)

## Changed

- The scaffold README now documents the custom icon convention: `icon = "warehouse"` loads
  `assets/icons/warehouse.svg`, while a nested path such as `icon = "app/warehouse"` maps to
  `assets/icons/app/warehouse.svg`. — Owner: [@Jixiangup](https://github.com/Jixiangup)
- Updated scaffold generation and application option tests to lock the application asset source,
  template rendering, and `assets/icons` directory generation behavior. — Owner:
  [@Jixiangup](https://github.com/Jixiangup)

## Issues and PRs

- Issue: none. This release comes from the desktop custom icon integration requirement.
- PR: none; committed and released directly from the default branch as requested.

## Compatibility and Breaking Changes

- Breaking changes: none.
- Existing applications that do not configure `application_assets` are unchanged and continue to use
  only Nexora and `gpui-component-assets` default resources.
- Newly generated projects add a `rust-embed` dependency and include an `assets/icons` directory that
  may be empty; `AppAssets` still compiles when the directory has no icons.
- The HTTP API, OpenAPI paths, ZITADEL integration, database schema, migrations, GPUI revision, and
  gpui-component revision are unchanged.

## Upgrade

1. Update the Nexora tag from `v0.6.0` to `v0.7.0` and refresh the lockfile.
2. Desktop applications that need custom icons should implement `AssetSource`, or follow the new
   scaffold's `AppAssets` pattern and place SVG files under `assets/icons/**/*.svg`.
3. Call `.application_assets(AppAssets)` in the application options. Existing projects that do not
   need custom icons do not need to change.
4. No database migration or server configuration change is required. Rolling back to `v0.6.0`
   removes the framework-level application asset registration entry point, but does not affect
   applications that never used it.

compilation.
