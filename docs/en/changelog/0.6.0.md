# Nexora 0.6.0

Release date: 2026-07-18

This early-alpha release adjusts the desktop Shell `SidebarHeader` slot semantics. After an
application declares `#[derive(nexora::SidebarHeader)]`, that custom implementation owns the full
Header content and is no longer rendered alongside the framework's default brand area. This affects
applications that already customize the Sidebar Header, so the release increments MINOR under
Semantic Versioning for the `0.y.z` line.

## Changed

- A custom `SidebarHeader` now replaces the default brand area. Without a custom Header, the Shell
  still shows the application name, logo, and subtitle from `ApplicationOptions`; with a custom
  Header, the Shell keeps only the structural boundary and divider while the application renders all
  content. — Owner: [@Jixiangup](https://github.com/Jixiangup)
- Updated the Sidebar Shell source-contract regression test so custom Header content must take
  priority over the default brand fallback, preventing the old append-after-brand behavior from
  returning. — Owner: [@Jixiangup](https://github.com/Jixiangup)
- Corrected documentation that still described custom Headers as being appended after the default
  brand. When brand and application context should both be visible, the custom Header should compose
  stable-ID `SidebarRegion` values directly. — Owner:
  [@Jixiangup](https://github.com/Jixiangup)

## Issues and PRs

- Issue: none. This release comes from the iMES desktop integration requirement that the Sidebar
  Header slot behaves as a replacement.
- PR: none; committed and released directly from the default branch as requested.

## Compatibility and Breaking Changes

- Breaking change: applications that declare a custom `#[derive(nexora::SidebarHeader)]` no longer
  get the default logo, application name, or subtitle automatically.
- Applications without a custom Header are unchanged and continue to render the default brand area.
- `SidebarFooter`, Feature, NavigationGroup, Window, SettingsWindow, and LoginFeature declarations
  and lifecycles are unchanged.
- The HTTP API, OpenAPI paths, ZITADEL integration, database schema, migrations, GPUI revision, and
  gpui-component revision are unchanged.

## Upgrade

1. Update the Nexora tag from `v0.5.2` to `v0.6.0` and refresh the lockfile.
2. If the application declares a custom `SidebarHeader`, check whether the logo, application name, or
   subtitle should still be shown.
3. To keep brand information visible, render it explicitly inside the custom Header. When separate
   hit regions are needed, compose stable-ID `nexora::desktop::SidebarRegion` values for brand,
   tenant, or factory selectors.
4. No database migration or server configuration change is required. Rolling back to `v0.5.2`
   restores the old behavior where the default brand and custom Header rendered together.
