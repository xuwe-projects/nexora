# Nexora 0.10.0

Release date: 2026-07-19

This release remains in the `0.x` beta line and does not move Nexora to `1.x`. The version uses the
standard SemVer value `0.10.0` without a beta suffix. Because this release changes the public
`FormDialog` API and the default top-tab visual style, it bumps the minor version in the `0.x` line.

## Added

- Added `nexora::desktop::{FormItem, FormItemControl}` for standard create/edit forms, including
  labels, descriptions, required markers, and built-in `input`, `password_input`, `number_input`,
  and `checkbox` controls. — Owner: [@Jixiangup](https://github.com/Jixiangup)
- Added a default-display setting to the built-in Settings window so the main window can reopen on a
  selected display; disconnected displays safely fall back to the system primary display. — Owner:
  [@Jixiangup](https://github.com/Jixiangup)
- Added standard CRUD Panel filtering and query actions to the default user-management and
  roles/permissions pages, including keyword, status, type, and role-kind filters. — Owner:
  [@Jixiangup](https://github.com/Jixiangup)

## Changed

- `FormDialog` now uses a builder API:
  `FormDialog::new(id, state).title(...).child(FormItem::new(...)).section(...)`, with
  `.with_size(theme::component_size(cx))` for the component-size setting. — Owner:
  [@Jixiangup](https://github.com/Jixiangup)
- The main-window Feature tabs now default to gpui-component's official default `Tabs` story style
  instead of `Segmented`; `ApplicationTabStyle` supports `Tab`, `Underline`, `Pill`, `Outline`, and
  `Segmented`. — Owner: [@Jixiangup](https://github.com/Jixiangup)
- `CrudPanel`, default Account buttons, filters, dropdowns, role editors, and user-create forms now
  consistently use `theme::component_size(cx)`. — Owner:
  [@Jixiangup](https://github.com/Jixiangup)
- Default create-user, create-role, and role-management dialogs now use business-facing copy instead
  of identity-provider implementation details; service accounts are displayed as a type and remain
  non-operable. — Owner: [@Jixiangup](https://github.com/Jixiangup)
- Generated-project `.agents/skills` and templates now document standard CRUD Panel, FormDialog,
  official tab styles, and component-size rules. — Owner:
  [@Jixiangup](https://github.com/Jixiangup)

## Fixed

- Fixed `PanelDialog` content layout so visible FormDialog content and footer buttons stay inside the
  dialog surface hit bounds. Create-user and create-role dialogs now render body content and keep
  cancel/submit clickable. — Owner: [@Jixiangup](https://github.com/Jixiangup)
- Fixed inconsistent refresh, query, and create button sizes in the default user and role pages. —
  Owner: [@Jixiangup](https://github.com/Jixiangup)

## Compatibility and Breaking Changes

- Breaking change: the old `FormDialog::new(id, state, title, content, on_submit)` constructor has
  been removed. Migrate to the builder API:

```rust
FormDialog::new("create-user-dialog", self.form.clone())
    .title("Create user")
    .description("Fill in the information to create a user.")
    .child(FormItem::new("Name").required().input(&self.name))
    .section(permission_list)
    .submit_label("Create user")
    .submit_disabled(!self.can_submit(cx))
    .with_size(theme::component_size(cx))
    .on_submit(cx.listener(Self::submit))
```

- Breaking behavior change: `PanelDialog` no longer closes on overlay clicks by default. Form dialogs
  should use cancel, the close button, or submit as explicit intents. Call `.overlay_closable(true)`
  when overlay-click dismissal is required.
- Behavior change: apps without an explicit `ApplicationOptions::tab_style(...)` move from the
  segmented visual to the official default `Tabs` visual. Set `ApplicationTabStyle::Segmented` to keep
  the old visual.
- No HTTP API, database migration, server configuration, or Account/ZITADEL contract change.

## Upgrade

1. Update the Nexora tag from `v0.9.1` to `v0.10.0` and refresh the lockfile.
2. Search for `FormDialog::new(` and migrate old constructor calls to the builder API with
   `FormItem` / `FormItemControl`.
3. If the application should keep the old segmented top-tab visual, explicitly set
   `ApplicationTabStyle::Segmented`; otherwise use the new official default `Tabs` style.
4. Sync the new generated `.agents/skills`, especially standard CRUD Panel, FormDialog builder,
   component-size, and official tab-style guidance. Standard CRUD business pages should prefer
   `nexora::desktop::{CrudPanel, CrudPanelToolbar}`.
5. Keep custom macro work on the validation loop: `cargo expand`, equivalent handwritten-code
   comparison, `cargo bench`, `cargo bloat`, and `cargo asm` analysis when benchmarks regress.
6. No database migration or server configuration change is required. Rolling back to `v0.9.1` also
   requires reverting FormDialog call sites.
