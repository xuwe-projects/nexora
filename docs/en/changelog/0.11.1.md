# Nexora 0.11.1

Release date: 2026-07-19

This release remains in the `0.x` beta line and does not move Nexora to `1.x`. The version uses the
standard SemVer value `0.11.1` without a beta suffix. This release only fixes and tightens the
default desktop management experience. It does not introduce incompatible public Rust API, HTTP
contract, configuration, database-migration, or scaffold-rule changes, so it is a patch release in
the `0.11.x` line.

## Fixed

- The default Roles & Permissions create form no longer shows a "role key" input to operators. The
  default client derives the hidden `CreateRoleRequest.key` from the role name and avoids keys
  already loaded in the current role list, while the server keeps the existing format validation and
  uniqueness constraint as the final guard. — Owner: [@Jixiangup](https://github.com/Jixiangup)
- The default role edit form no longer shows the read-only role-key field, so the authorization
  identifier is no longer presented as editable business form information. The role list still keeps
  the key as read-only identifying metadata. — Owner: [@Jixiangup](https://github.com/Jixiangup)
- `PanelDialog` now scrolls its body inside the dialog surface, and `FormDialog` defaults to 80% of
  the current Feature Panel height with a responsive width cap. Large forms no longer push the title
  or footer out of the panel. — Owner: [@Jixiangup](https://github.com/Jixiangup)

## Issues and PRs

- Issue: none. This release comes from default Account role-management form and FormDialog responsive
  layout fixes.
- PR: none; committed and released directly from the default branch as requested.

## Compatibility and Breaking Changes

- No public API breaking changes. The `CreateRoleRequest.key` contract is unchanged; the only change
  is in Nexora's default desktop role-create page, where users no longer type that field manually.
- No database migration, OpenAPI route, server configuration, or generated-project structure change.
- Rolling back to `v0.11.0` only restores the old default role form and old FormDialog height
  behavior. No data migration is involved.

## Upgrade

1. Update the Nexora tag from `v0.11.0` to `v0.11.1` and refresh the lockfile.
2. Desktop apps using the default Account role-management page do not need code changes. Operators
   continue to fill in the role name, description, and permissions.
3. Apps using `nexora::desktop::FormDialog` directly automatically receive the new responsive height
   and body-scroll defaults.
4. No generated-project Skill sync, macro-rule update, or database migration is required. This
   release does not change generated-project constraints.
