# Nexora 0.11.2

Release date: 2026-07-19

This release remains in the `0.x` beta line and does not move Nexora to `1.x`. The version uses the
standard SemVer value `0.11.2` without a beta suffix. This is a patch correction for `0.11.1` and
only changes the default desktop `FormDialog` height semantics and release notes. It does not
introduce incompatible public Rust API, HTTP contract, configuration, database-migration, or
scaffold-rule changes.

## Fixed

- The regular create/edit `FormDialog` surface now defaults to exactly 80% of the current Feature
  Panel height instead of only setting a maximum height. The title and footer stay fixed, and the
  form body continues to scroll independently. — Owner: [@Jixiangup](https://github.com/Jixiangup)
- `PanelDialog` overlay no longer shrinks the percentage-height reference with padding, so
  `relative(0.8)` is now calculated against the Panel itself. Dialogs no longer end up slightly
  shorter than 80% of the Panel. — Owner: [@Jixiangup](https://github.com/Jixiangup)

## Added

- `FormDialog::panel_height_ratio(ratio)` lets applications adjust the regular form height ratio
  relative to the Panel. The default is `0.8`.
- `FormDialog::auto_height()` lets very small forms return to content-sized height while preserving
  the default 80% Panel-height cap. Large forms still scroll in the body area.

## Issues and PRs

- Issue: none. This release corrects the `FormDialog` Panel-height semantics after `0.11.1`.
- PR: none; committed and released directly from the default branch as requested.

## Compatibility and Breaking Changes

- No public API breaking changes. The default `FormDialog` visual height now becomes 80% of the
  current Panel; this fixes the incorrect `0.11.1` default. Pages that truly need the old
  content-sized behavior can call `FormDialog::auto_height()` explicitly.
- No database migration, OpenAPI route, server configuration, or generated-project structure change.
- Rolling back to `v0.11.1` restores the max-height-only behavior. No data migration is involved.

## Upgrade

1. Update the Nexora tag from `v0.11.1` to `v0.11.2` and refresh the lockfile.
2. Apps using `nexora::desktop::FormDialog` automatically receive the corrected 80% Panel-height
   default.
3. Only pages that intentionally want a content-sized small form need to append
   `FormDialog::auto_height()` when building the form.
4. No generated-project Skill sync, macro-rule update, or database migration is required. This
   release does not change generated-project constraints.
