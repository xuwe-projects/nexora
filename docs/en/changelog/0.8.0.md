# Nexora 0.8.0

Release date: 2026-07-19

This early-alpha release adds standard Panel components for desktop CRUD resource management and
fixes several layout and dialog issues in the default Account user and role management pages.
Because GPUI and gpui-component are still consumed through pinned git revisions, Nexora continues to
ship `0.x` prerelease versions and does not enter the `1.x` stable line yet.

## Added

- Added `nexora::desktop::{CrudPanel, CrudPanelToolbar}` for standard CRUD Feature Panels: title
  summary, refresh button, optional filter/action areas, and a data body that fills the remaining
  height by default. The toolbar card is omitted when no filters or actions are provided. — Owner:
  [@Jixiangup](https://github.com/Jixiangup)
- Added `nexora::desktop::TableHeaderCell` for DataTable headers that are horizontally and
  vertically centered by default while still allowing `.left()`, `.center()`, `.right()`, or fully
  custom header elements. — Owner: [@Jixiangup](https://github.com/Jixiangup)
- Added UI crate tests covering empty-toolbar omission, toolbar content detection, and default
  centered/customizable table headers. — Owner: [@Jixiangup](https://github.com/Jixiangup)

## Changed

- The default user management page now uses `CrudPanel` for its title summary, refresh action,
  create-user action, and users table; the table body still fills the remaining height and keeps
  scrolling/infinite loading inside the DataTable. — Owner:
  [@Jixiangup](https://github.com/Jixiangup)
- Updated generated project `.agents/skills` to recommend `CrudPanel` / `CrudPanelToolbar` for
  standard CRUD Panels and `TableHeaderCell` for DataTable headers. — Owner:
  [@Jixiangup](https://github.com/Jixiangup)

## Fixed

- Fixed truncated content in the default users table user column by widening the column and keeping
  display name and ID truncation stable within constrained widths. — Owner:
  [@Jixiangup](https://github.com/Jixiangup)
- Fixed the users table status column being too wide; it now uses a compact fixed width suited for
  short status labels. — Owner: [@Jixiangup](https://github.com/Jixiangup)
- Fixed double scrolling between the default users page and DataTable by disabling Shell content
  scrolling for that Feature and letting DataTable own the scroll viewport. — Owner:
  [@Jixiangup](https://github.com/Jixiangup)
- Fixed default user and role create/edit overlays not covering the current Feature Panel; default
  Account overlays now use a stable panel-scoped overlay. — Owner:
  [@Jixiangup](https://github.com/Jixiangup)
- Fixed the default role management refresh button icon so it matches user management and uses
  `rotate-ccw.svg`. — Owner: [@Jixiangup](https://github.com/Jixiangup)

## Issues and PRs

- Issue: none. This release comes from the default Account management UX and standard CRUD Panel
  extraction requirement.
- PR: none; committed and released directly from the default branch as requested.

## Compatibility and Breaking Changes

- Breaking changes: none.
- The new `CrudPanel`, `CrudPanelToolbar`, and `TableHeaderCell` APIs are backward-compatible;
  existing pages may keep their custom layouts.
- The HTTP API, OpenAPI paths, ZITADEL integration, database schema, migrations, GPUI revision, and
  gpui-component revision are unchanged.
- This release remains a `0.x` prerelease. Nexora will not move to the `1.x` stable line before GPUI
  and gpui-component are published as crates.

## Upgrade

1. Update the Nexora tag from `v0.7.0` to `v0.8.0` and refresh the lockfile.
2. Existing projects should sync the new CRUD Panel guidance from the generated `.agents/skills`;
   newly created projects include it automatically.
3. Resource-management pages with a title summary, refresh, filters/actions, and data body should
   consider `nexora::desktop::{CrudPanel, CrudPanelToolbar}`; DataTable headers should consider
   `nexora::desktop::TableHeaderCell`.
4. After upgrading, run `cargo clean` locally before the next `cargo check` or app run to avoid
   stale GPUI build artifacts while diagnosing issues.
5. No database migration or server configuration change is required. Rolling back to `v0.7.0`
   removes the new CRUD Panel components but does not affect applications that never used them.
