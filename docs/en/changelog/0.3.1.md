# Nexora 0.3.1

Released on 2026-07-18.

This patch exposes a token-free Account authentication scope and observer to desktop hosts. An
application can now clear its own session-bound Globals immediately on sign-out, re-login,
authenticator replacement, or account change, and can use the scope revision to prevent an old
asynchronous response from writing into a new session without accessing Nexora internals.

## Added

- Added `AccountAuthenticationScope { revision, user_id }`. `user_id` is the current local
  `account.users(id)` value and is `None` while signed out. The snapshot contains no access token,
  refresh token, or provider details. — Owner: [@Jixiangup](https://github.com/Jixiangup)
- Added `nexora::desktop::authentication_scope(&App)` so a host can capture the current safe scope
  alongside a business request and compare it before applying the result. — Owner:
  [@Jixiangup](https://github.com/Jixiangup)
- Added `nexora::desktop::observe_authentication(&mut Context<T>, callback)`. Its `Subscription` is
  owned by the host Entity and fires only when the authentication scope changes, not for login
  status text or busy-state updates. — Owner: [@Jixiangup](https://github.com/Jixiangup)

## Changed

- Account now tracks the authentication-scope revision separately from the browser-login task
  generation. Committing either a successful or failed login result, signing out, and replacing the
  authenticator invalidate the old scope; merely starting a login does not prematurely clear an
  authenticated business session. Existing Shell gates and Feature and Sidebar lifecycles are
  unchanged. — Owner: [@Jixiangup](https://github.com/Jixiangup)
- Workspace component versions are synchronized at 0.3.1. Account server behavior, the default user
  and role management pages, and database migrations remain compatible with 0.3.0, allowing hosts
  to pin this desktop addition with one tag. — Owner: [@Jixiangup](https://github.com/Jixiangup)

## Compatibility and breaking changes

- This release only adds public desktop APIs and does not remove or change any 0.3.0 API. It is a
  backward-compatible patch.
- It adds, removes, or changes no database DDL and needs no data or `_sqlx_migrations` migration.
- The new `user_id` is the Nexora-local `account.users(id)`, not an OIDC subject, and must not be
  treated as a cross-deployment or cross-process identifier.
- `revision` only isolates concurrent work in the current process. Do not persist it or compare it
  after an application restart.

## Upgrade from 0.3.0

1. Pin Nexora to `v0.3.1` and update the lockfile:

   ```toml
   nexora = { git = "https://github.com/xuwe-projects/nexora", tag = "v0.3.1", default-features = false }
   ```

   ```bash
   cargo update -p nexora
   ```

2. Store the observer in a long-lived Entity when a desktop application owns session-bound business
   Globals:

   ```rust
   let initial = nexora::desktop::authentication_scope(cx);
   let subscription = nexora::desktop::observe_authentication(cx, |this, scope, cx| {
       this.authentication_changed(scope, cx);
   });
   ```

3. Capture `AccountAuthenticationScope` for each asynchronous business request and discard the
   result if it no longer matches the current scope. A business client receiving HTTP 401 should
   still call `nexora::desktop::sign_out(cx)` so the common gate and all host observers clear state;
   HTTP 403 only means insufficient permission and must not sign out.
4. Reinstall the 0.3.1 CLI:

   ```bash
   cargo install --git https://github.com/xuwe-projects/nexora --tag v0.3.1 nexora --locked --force --no-default-features --features cli --bin nexora
   ```

## Rollback notes

- There is no database change and no down migration.
- Before returning to 0.3.0, remove host calls to `AccountAuthenticationScope`,
  `authentication_scope`, and `observe_authentication`, then restore the dependency tag and
  lockfile.
- An application that relies on these APIs to clear business Globals cannot roll back Nexora alone;
  roll back the corresponding host code as well or it will fail to compile.
