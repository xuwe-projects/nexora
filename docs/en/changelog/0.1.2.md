# Nexora 0.1.2

Released on 2026-07-17.

This release improves product documentation and establishes a repeatable GitHub release process.
It introduces no breaking framework API changes.

## Added

- Added a responsive product overview that combines the original workspace, settings, and login
  screenshots into three light/dark cards without redrawing UI text. — Owner:
  [@Jixiangup](https://github.com/Jixiangup)
- Added the `publish-nexora-release` Skill for version inventory, owner and issue attribution,
  upgrade notes, validation, tags, and GitHub Releases. Generated projects receive the Skill. —
  Owner: [@Jixiangup](https://github.com/Jixiangup)
- Added an Ubuntu and Windows CLI installation matrix: branch runs verify local-source installs,
  release tags verify GitHub-tag installs, and both run `nexora --version`. — Owner:
  [@Jixiangup](https://github.com/Jixiangup)

## Changed

- Moved complete release notes and embedded component changelogs into the VitePress documentation.
  — Owner: [@Jixiangup](https://github.com/Jixiangup)
- Documented both local-source and released GitHub-tag installation with single-line commands that
  work in Bash, zsh, PowerShell, and CMD. — Owner:
  [@Jixiangup](https://github.com/Jixiangup)

## Fixed

- Corrected the homepage description of Server ownership: the host owns the pool, listener, and
  service lifecycle. — Owner: [@Jixiangup](https://github.com/Jixiangup)

## Compatibility

There are no public API, configuration, or database breaking changes from `0.1.1` to `0.1.2`.

## Upgrade from 0.1.1

Reinstall the released CLI:

```bash
cargo install --git https://github.com/xuwe-projects/nexora --tag v0.1.2 --locked --force --no-default-features --features cli --bin nexora
```

Update the Nexora workspace dependency tag and refresh the lockfile:

```toml
nexora = { git = "https://github.com/xuwe-projects/nexora", tag = "v0.1.2", default-features = false }
```

```bash
cargo update -p nexora
```

Run `nexora init` with the project's existing layout to add the release Skill to an existing
project. The command adds missing files without replacing existing rules or Skills.
