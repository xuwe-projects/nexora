# Nexora 0.5.2

Release date: 2026-07-18

This patch fixes the default application Shell's Sidebar navigation rendering. The Shell now uses
the official `gpui-component` `Sidebar`, `SidebarGroup`, `SidebarMenu`, and `SidebarMenuItem`
components again, preventing navigation labels from being centered by custom button layout. Public
APIs, configuration, database migrations, and dependency revisions remain compatible with 0.5.1, so
the release increments PATCH under Semantic Versioning.

## Fixed

- The default framework Sidebar navigation now renders sections, folders, and Feature entries with
  the official `gpui_component::sidebar` component family. Navigation labels are back to the
  official `SidebarMenuItem` left-aligned behavior while active, hover, and expandable folder states
  stay owned by the component library. — Owner: [@Jixiangup](https://github.com/Jixiangup)
- Preserved the custom `SidebarHeader` and `SidebarFooter` slot boundaries: the Shell only owns
  structure, spacing, and dividers, and does not inject hover, selected, radius, cursor, or click
  semantics into custom slot content. — Owner: [@Jixiangup](https://github.com/Jixiangup)
- Added a source-contract regression test requiring the main Shell to use the official Sidebar,
  Group, Menu, and MenuItem components so custom Button or flex navigation does not return. —
  Owner: [@Jixiangup](https://github.com/Jixiangup)
- Updated the docs build script to start VitePress explicitly through Node, avoiding a local release
  environment SIGKILL when Bun executes the `vitepress` bin directly. — Owner:
  [@Jixiangup](https://github.com/Jixiangup)

## Issues and PRs

- Issue: none. This fixes a Sidebar alignment problem found while integrating the iMES desktop app.
- PR: none; fixed directly on the default branch for this release.

## Compatibility

There are no breaking changes and no caller migration is required. Feature, NavigationGroup,
SidebarHeader, and SidebarFooter declarations remain unchanged. The HTTP API, OpenAPI paths,
ZITADEL integration, database schema, migrations, GPUI revision, gpui-component revision, and assets
revision are unchanged. The generated docs content is unchanged; only the `docs/package.json` build
script startup path changed.

## Upgrade

Update the Nexora tag from `v0.5.1` to `v0.5.2` and refresh the lockfile. No database migration or
configuration change is required. Rolling back only requires restoring the `v0.5.1` dependency, but
that reintroduces the default Sidebar navigation label alignment issue.
