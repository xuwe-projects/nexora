# Nexora 0.9.0

Release date: 2026-07-19

This release remains in the `0.x` beta line and does not move Nexora to `1.x`. Until GPUI and
gpui-component are published as crates, Nexora continues to iterate as `0.x`. This version extends
the standard CRUD Panel work with reusable `CrudTableRow` / `CrudTableDelegate` table helpers,
polishes the default Account management pages, and exposes the top tab style as public API.

## Compatibility and Breaking Changes

- Breaking change: `UserResponse` JSON now includes required `user_type`; strict clients must update
  their DTOs.
- Breaking change: standard CRUD table guidance moves from handwritten `TableDelegate` boilerplate to
  `CrudTableRow` / `CrudTableDelegate`. Existing pages may still implement the native `TableDelegate`,
  but new scaffolds and skills prefer the new API.
- Breaking change: service accounts can no longer be mutated through default Account status or
  direct-role APIs; those requests return `409 service_account_immutable`.
- The top tab visual style now defaults to gpui-component's official `Segmented` variant; applications
  may use `ApplicationOptions::tab_style` to choose another official style.
- The database schema, migrations, GPUI revision, and gpui-component revision are unchanged.
- This release remains in the `0.x` beta line. Nexora will not move to `1.x` before GPUI and
  gpui-component are published as crates.

## Upgrade

1. Update the Nexora tag from `v0.8.0` to `v0.9.0` and refresh the lockfile.
2. Sync the new generated `.agents/skills`, especially the standard CRUD Panel, `CrudTableRow`, top
   tab style, and custom macro validation loop rules.
3. If a client strictly deserializes `User` / `UserResponse`, add
   `user_type: "human" | "service_account"`.
4. Standard resource-management pages should use `nexora::desktop::{CrudPanel, CrudPanelToolbar}`;
   CRUD tables should use `#[derive(nexora::CrudTableRow)]` + `CrudTableDelegate<T>`, adding operation
   columns with `action_column`.
5. If an application wants a different tab visual style, set
   `ApplicationOptions::tab_style(ApplicationTabStyle::Underline | Pill | Outline)`.
6. After upgrading, run `cargo clean` locally before the next `cargo check` or app launch to avoid
   stale GPUI build artifacts while diagnosing issues.
7. No database migration or server configuration change is required. Rolling back to `v0.8.0`
   removes the new table helpers and the `user_type` response field.
