# Nexora 0.9.1

Release date: 2026-07-19

This release remains in the `0.x` beta line and does not move Nexora to `1.x`. The version uses the
standard SemVer value `0.9.1` without a beta suffix. This patch fixes the top window tab bar when
`Segmented` styling rendered pinned and regular tabs as two separate `TabBar` controls, which made
the segmented surface look visually split. It also fixes a GPUI Entity re-entrancy panic that could
occur when the default user management page was activated.

## Fixed

- The top Feature tab bar now renders through one gpui-component `TabBar`. Pinned tabs still stay at
  the front of the ordered tab list, but the segmented background, rounded container, and sliding
  indicator now remain one continuous control instead of two adjacent tab bars. — Owner:
  [@Jixiangup](https://github.com/Jixiangup)
- The top tab scroll state now uses one `ScrollHandle` for the official `TabBar` horizontal scroll
  area. Tab selection, context menus, closing, and unpinning behavior remain unchanged. — Owner:
  [@Jixiangup](https://github.com/Jixiangup)
- The default user management page no longer lets the child table read `UsersPage` while that parent
  Entity is being updated during the initial user load. This fixes
  `cannot read ... UsersPage while it is already being updated` when opening “用户管理” in an
  authenticated app. — Owner: [@Jixiangup](https://github.com/Jixiangup)

## Issues and PRs

- Issue: none. This release comes from top tab bar visual continuity and default user management
  crash fixes.
- PR: none; committed and released directly from the default branch as requested.

## Compatibility and Breaking Changes

- No breaking changes.
- No HTTP API, database migration, configuration, ZITADEL/Account contract, or scaffold structure
  change.
- Pinned top tabs keep the semantic meaning of being ordered at the front of the tab list; pinned
  tabs no longer have a separate scroll region.
- The default user management fix only changes internal GPUI Entity data flow and does not change
  user, role, or permission APIs.
- This release does not change `.agents/skills`, macro rules, or generated-project constraints.
  Downstream projects do not need to sync skills for this upgrade.

## Upgrade

1. Update the Nexora tag from `v0.9.0` to `v0.9.1` and refresh the lockfile.
2. If the application uses the default `ApplicationTabStyle::Segmented`, the top tab bar will
   automatically use the connected single-`TabBar` visual.
3. If the application explicitly uses another official style through
   `ApplicationOptions::tab_style(...)`, no code change is required.
4. No database migration, server configuration update, or `.agents/skills` sync is required.
5. Rolling back to `v0.9.0` only restores the old two-`TabBar` top tab layout.
