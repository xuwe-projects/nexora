# Nexora 0.4.1

Released on 2026-07-18.

This patch fixes the machine-readable OpenAPI contract missing from the 0.4.0 VitePress deployment.
The published `v0.4.0` tag and Release remain immutable; strict Semantic Versioning advances PATCH
to 0.4.1.

## Fixed

- Moved the single maintained OpenAPI 3.1 source into VitePress `public`, so every build publishes
  `/openapi.yaml` and links from the Chinese and English HTTP/Rust server references no longer
  return 404. — Owner: [@Jixiangup](https://github.com/Jixiangup)
- Updated configuration documentation to point to `docs/public/openapi.yaml` in the repository. —
  Owner: [@Jixiangup](https://github.com/Jixiangup)

## Related issues and pull requests

- Issues: none. Post-release Pages accessibility verification found the problem.
- Pull requests: none. This release does not invent or associate an unrelated pull request.

## Compatibility

- Rust, HTTP, database, and configuration contracts are unchanged.
- OpenAPI operations are unchanged from 0.4.0; only `info.version`, the source location, and the
  Pages artifact change.
- This backward-compatible documentation deployment fix advances PATCH and needs no migration.

## Upgrade from 0.4.0

1. Pin Nexora to `v0.4.1` and update the lockfile:

   ```toml
   nexora = { git = "https://github.com/xuwe-projects/nexora", tag = "v0.4.1", default-features = false }
   ```

2. No new migration or source change is required.
3. Download the machine-readable contract from
   `https://xuwe-projects.github.io/nexora/openapi.yaml`.

## Rollback notes

- Rolling back to 0.4.0 changes no database or runtime code, but `/openapi.yaml` becomes unavailable
  on Pages again.
- Never move or rewrite the published `v0.4.0` and `v0.4.1` tags.
