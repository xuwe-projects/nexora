# Nexora 0.5.0

Release date: 2026-07-18

This release adds first-class navigation-only groups, panel-scoped form dialogs, independent Sidebar
regions, and a Cascader, while closing the ZITADEL user-management loop. Removing page Features as
navigation parents is a breaking public-model change, so the early-development SemVer line advances
from 0.4.1 to 0.5.0.

## Added

- `#[derive(nexora::NavigationGroup)]` for recursive, non-routable Sidebar directories with stable
  ordering, active-ancestor expansion, and non-clickable breadcrumb labels. — Owner:
  [@Jixiangup](https://github.com/Jixiangup)
- `FormDialog`, `FormDialogState`, and `FormFieldDraft` for panel-only create/edit forms, scrollable
  content, required submit handling, and default unsaved-draft confirmation. — Owner:
  [@Jixiangup](https://github.com/Jixiangup)
- A gpui-component-composed single-select Cascader with stable value paths, search, disabled options,
  clear, separators, and `change_on_select`. — Owner: [@Jixiangup](https://github.com/Jixiangup)
- Stable-ID `SidebarRegion`, migration 7 for administrator-permission backfill, GPUI revision linting,
  and a clean-lock scaffold consumer check. — Owner: [@Jixiangup](https://github.com/Jixiangup)

## Changed

- Feature `parent` is removed. A page Feature uses `group` to reference a NavigationGroup; group
  parents may only reference groups in the same section. Duplicate IDs, unknown references, cycles,
  self references, and cross-section edges are rejected. — Owner: [@Jixiangup](https://github.com/Jixiangup)
- Custom Sidebar Header/Footer slots no longer receive Shell hover, selection, radius, cursor, or
  click behavior. The default brand remains visible and application context is appended separately.
  — Owner: [@Jixiangup](https://github.com/Jixiangup)
- The users table is content-height, card-surfaced, continuously loaded, column-movable, and
  width-resizable. It shows avatars and compact status tags while hiding identity IDs. User and role
  create/edit flows now use panel-scoped FormDialog. — Owner: [@Jixiangup](https://github.com/Jixiangup)
- `POST /users` now creates a ZITADEL human user from username, given/family names, and email before
  binding the returned identity ID locally. `GET /me` refreshes Provider profile data. OIDC server
  configuration now requires `organization_id`. — Owner: [@Jixiangup](https://github.com/Jixiangup)
- Newly registered permissions are granted to the system-administrator role in the same transaction.
  — Owner: [@Jixiangup](https://github.com/Jixiangup)
- Verified dependency matrix: gpui-component/assets
  `031555662e99a1b5a549990b47f246d475b8288a` with Zed/GPUI
  `1a246efd7e1b83ab568ec5e3e6c1a43a42e1abba`. Templates pin both sources and replace the upstream
  unpinned gpui/gpui_macros edges. — Owner: [@Jixiangup](https://github.com/Jixiangup)

## Upgrade from 0.4.1

1. Replace every Feature `parent = "id"` with a declared `NavigationGroup` and leaf Feature
   `group = "id"`; do not keep placeholder pages or a second navigation table.
2. Add `[oidc].organization_id`. Update user-create clients to send `username`, `given_name`,
   `family_name`, `email`, optional `display_name`, and `role_ids`; remove `identity_id` and
   `avatar_url`.
3. Run the single merged SQLx Migrator, including migration 7, before deploying the server.
4. Move custom Header/Footer interaction styles into stable-ID `SidebarRegion` values.
5. Delete the downstream lockfile that captured the incompatible source pair and re-lock with the
   pinned gpui-component and GPUI revisions. Run Nexora lint and a clean-lock `cargo check`.

Migration 7 has a safe no-op down migration because backfilled administrator grants cannot be
distinguished from earlier legitimate grants. Restore a pre-upgrade database backup for a full
rollback. Provider users created after the upgrade are not automatically removed by a code rollback.
