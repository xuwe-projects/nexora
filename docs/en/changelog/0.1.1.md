# Nexora 0.1.1

Released on 2026-07-17.

This release completed the default full-stack Account experience and reassigned resource ownership
between Nexora and host applications.

## Added

- Added Setup initialization, ZITADEL human-user selection, and super-administrator binding. —
  Owner: [@Jixiangup](https://github.com/Jixiangup)
- Added user, role, permission, and authorization management plus default desktop pages. — Owner:
  [@Jixiangup](https://github.com/Jixiangup)
- Added application-branded login UI and request-ID notification actions. — Owner:
  [@Jixiangup](https://github.com/Jixiangup)
- Added bilingual VitePress documentation and GitHub Pages deployment. — Owner:
  [@Jixiangup](https://github.com/Jixiangup)

## Changed

- Server now handles migrations, Account/ZITADEL initialization, and router composition only; the
  host owns its pool, listener, Axum state, logging, and shutdown policy. — Owner:
  [@Jixiangup](https://github.com/Jixiangup)
- Desktop authentication moved to `nexora::desktop`, while server configuration and extension
  points moved to `nexora::server`. — Owner: [@Jixiangup](https://github.com/Jixiangup)
- SQLx migration history became the only upgrade gate. — Owner:
  [@Jixiangup](https://github.com/Jixiangup)

## Related issues and pull requests

None.

## Breaking changes and upgrade from 0.1.0

1. Use only `desktop,derive` for desktop packages and replace `nexora::account::client::*` with
   `nexora::desktop::*`.
2. Use only `server,derive` for servers. Construct `Server::new()`, provide the application pool to
   `initialize`, merge `server.routers()`, and start with standard `axum::serve`.
3. Split `server.bind` into `server.ip` and `server.port`, add the Setup secret and ZITADEL project
   credentials, and remove `initialize_empty_database`.
4. Back up production data and configuration before upgrading. SQLx skips versions already recorded
   as successfully applied.
